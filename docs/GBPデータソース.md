# GBP 現在データのソース

月次パフォーマンス（ユーザー／電話／ルート検索／WEBサイト）の**現在のデータ**は、以下の Excel ファイルの **GBPサマリー** シートを正とする。

---

## ファイル・シート

| 項目 | 内容 |
|------|------|
| **ファイル** | `tmp/Googleビジネスプロフィール集計.xlsx` |
| **シート名** | **GBPサマリー** |
| **備考** | 本ファイルはリポジトリの `tmp/` に配置。機密・社外非公開の場合は .gitignore で除外すること。 |

---

## GBPサマリー シートの構成（確認済み）

- **行数**: 152 行（ヘッダ＋店舗ブロック）
- **列**: A〜（未使用）, B=店舗, C=指標, D〜=月（日付型の月初日）

**ヘッダ（2行目）**

- B: 店舗  
- C: 指標  
- D 以降: 月（2023年7月 〜 2026年2月、32ヶ月分）

**データ（3行目〜）**

- 店舗ごとに **5行のブロック**: ユーザー / 電話 / ルート検索 / WEBサイト / 合計  
- **B列は店舗ごとに 5 行で結合**されている。店舗名はその結合セル（ブロック先頭）に記載。値の取得時は先頭行の B にのみ現れる。  
- 指標ごとの数値は C 列以右の月列に配置。

**シート内の店舗名（30店舗、確認時点）**

川越, 平塚, 越谷, 江戸川, 松戸, 東村山, 幕張, 川崎丸子橋, 町田, 八王子, 習志野, さいたま見沼, 新座, 前橋, 摂津, 箕面, 藤沢, 福岡志免, 相模原, 浜松, 羽村, 岡山, 青梅, 札幌西野, 八王子高倉, 高崎, 名古屋北, 札幌清田, 高槻, 藤岡

※ 表記が dim_store と異なる場合あり（例: さいたま見沼 ↔ 見沼、福岡志免 ↔ 志免）。インポート時に店舗マスタ [店舗マスタ参照.md](店舗マスタ参照.md) で store_id と対応させる。

## xlsx からの upsert（推奨）

**GBPサマリー** の内容をそのまま BigQuery に反映する場合は、以下のスクリプトで **MERGE（upsert）** を実行する。

```bash
python scripts/import_gbp_monthly_from_xlsx.py
```

- シートを読み、店舗名を [店舗マスタ参照](店舗マスタ参照.md) の store_code にマッピングしてから `performance_monthly_snapshot` に MERGE。
- 既存行は上書き、存在しない (snapshot_month, store_code, provider) は INSERT。
- オプション: `--dry-run`（SQL のみ出力）、`--xlsx`（ファイル指定）、`--project` / `--dataset`（BQ 先）。生成 SQL は `sql/030_merge_monthly_from_xlsx.sql`。
