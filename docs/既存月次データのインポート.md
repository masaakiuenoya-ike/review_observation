# 既存月次データのインポート

店舗×月次の既存集計（ユーザー／電話／ルート検索／WEBサイト）を BigQuery に取り込む手順とタイミング。

**現在のデータの出所**: [tmp/Googleビジネスプロフィール集計.xlsx](../tmp/Googleビジネスプロフィール集計.xlsx) の **GBPサマリー** シート（詳細は [GBPデータソース.md](GBPデータソース.md)）。

---

## 1. 対象テーブルとタイミング

| 項目 | 内容 |
|------|------|
| **投入先** | **performance_monthly_snapshot** のみ（月次テーブル）。日次（performance_daily_snapshot）は今回のインポート対象外（日次の過去データがなければ空のままでよい）。 |
| **実施タイミング** | **places_provider_map に dim_store の全店舗を登録した直後** がよい。store_code が決まっていないと INSERT できないため。 |
| **対象店舗** | **dim_store に存在する行は全てインポート対象**。SQL は全 store_id について 2023-07/08/09 の行を挿入する（川越・平塚・越谷・江戸川は実データ、それ以外は metrics を NULL）。 |

**手順の順序**

1. **store_code** は店舗マスタ **ikeuchi-ga4.stg_freee_prd.dim_store** の **store_id** を文字列にした値を使う。川越・平塚・越谷・江戸川の対応は [docs/店舗マスタ参照.md](店舗マスタ参照.md) を参照。
2. **places_provider_map** に **dim_store の全店舗**を 1 行ずつ INSERT（store_code = store_id の文字列、provider='google', provider_place_id, is_active=true）。
3. 続けて **既存月次データ** を performance_monthly_snapshot に INSERT（`sql/020_import_historical_monthly_performance.sql` を実行。全店舗分は `scripts/gen_020_import_monthly.py` で再生成可）。

---

## 2. 列の対応

既存データと performance_monthly_snapshot の対応は次のとおり。

| 既存データ | テーブル列 | 備考 |
|------------|------------|------|
| 店舗（川越 等） | store_code | 店舗マスタ dim_store の store_id を文字列にした値（例: 川越→`3547880`）。[店舗マスタ参照.md](店舗マスタ参照.md) 参照。 |
| 2023年7月 等 | snapshot_month | DATE の月初日。例: 2023-07-01 |
| ユーザー | impressions | 空なら NULL |
| 電話 | calls | そのまま |
| ルート検索 | direction_requests | そのまま |
| WEBサイト | website_clicks | そのまま |
| （なし） | provider | 固定値 `'google'` |
| （なし） | provider_place_id | 空でよい（NULL）または places_provider_map の値 |
| （なし） | fetched_at, ingest_run_id, status | インポート用に固定値でよい（例: fetched_at=CURRENT_TIMESTAMP(), ingest_run_id='import', status='ok'） |

---

## 3. 具体的な処理方法（SQL）

- 投入先は **mart_gbp.performance_monthly_snapshot**。**dim_store に存在する行は全てインポート対象**とする。
- 実データがあるのは川越・平塚・越谷・江戸川の 4 店舗（2023-07/08/09）のみ。それ以外の店舗は同期間で impressions/calls/direction_requests/website_clicks を NULL で挿入する。
- 本体 SQL は **sql/020_import_historical_monthly_performance.sql**。dim_store の店舗数が変わった場合は **scripts/gen_020_import_monthly.py** の `STORE_IDS` を `ikeuchi-ga4.stg_freee_prd.dim_store` の最新一覧に合わせて更新し、以下で SQL を再生成する。

```bash
python scripts/gen_020_import_monthly.py > /tmp/values.txt
# sql/020 の ") VALUES" の直後を /tmp/values.txt の内容で置き換え、末尾に ";" を付与
```

- ユーザー（impressions）は既存データが空のため NULL。実データを追加する場合はスクリプトの `REAL_DATA` に店舗ごとの値を追加する。

---

## 4. 実行方法

- **bq query** で実行する場合（1 文にまとめて実行）:

```bash
bq query --project_id=ikeuchi-ga4 --location=asia-northeast1 --use_legacy_sql=false \
  "$(cat sql/020_import_historical_monthly_performance.sql)"
```

- または、上記 SQL を `sql/020_import_historical_monthly_performance.sql` に保存し、**BigQuery コンソール** で開いて実行してもよい。

---

## 5. 日次テーブルについて

- **performance_daily_snapshot** には、今回のような「既存の月次集計」は入れない。
- 日次は **これ以降**、Cloud Run の日次ジョブ（GBP API 取得）でだけ投入する想定。
- 過去に日次の実データがある場合のみ、別途「日次用のインポート」を検討する。
